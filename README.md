# Fridge Meal Planner documentation

This folder separates what the prototype currently implements from the target
product design so the team can build without confusing planned features with
available behaviour.

| Document | Purpose |
|---|---|
| [Product and build plan](PRODUCT_AND_BUILD_PLAN.md) | Product scope, MVP, delivery phases, and acceptance criteria |
| [Tech stack and architecture](TECH_STACK_AND_ARCHITECTURE.md) | Mobile, C# backend, PostgreSQL, AI layer, deployment, and C# trade-offs |
| [User flows](USER_FLOWS.md) | End-to-end flows for onboarding, receipt intake, fridge, planning, shopping, and AI assistance |
| [Backend, AI, and API](BACKEND_AI_AND_API.md) | Service boundaries, AI tool pattern, verification rules, and current endpoints |
| [Data schema](DATA_SCHEMA.md) | Implementation-aligned relational graph, tables, constraints, and schema gaps |
| [Implementation audit](IMPLEMENTATION_AUDIT.md) | Current capability map, integration mismatches, and prioritized fixes |
| [Project documentation (Word)](PROJECT_DOCUMENTATION.docx) | Editable consolidated project handbook |

## Status labels

- **Implemented:** present in the checked-in prototype.
- **Partial:** present but incomplete or not end-to-end.
- **Planned:** recommended target design; not yet available.
